114 """
115 @macroexpand
116
117 Return equivalent expression with all macros removed (expanded).
118
119 There are differences between `@macroexpand` and [`macroexpand`](@ref).
120
121 * While [`macroexpand`](@ref) takes a keyword argument `recursive`, `@macroexpand`
122 is always recursive. For a non recursive macro version, see [`@macroexpand1`](@ref).
123
124 * While [`macroexpand`](@ref) has an explicit `module` argument, `@macroexpand` always
125 expands with respect to the module in which it is called.
126
127 This is best seen in the following example:
128 ```julia-repl
129 julia> module M
130 macro m()
131 1
132 end
133 function f()
134 (@macroexpand(@m),
135 macroexpand(M, :(@m)),
136 macroexpand(Main, :(@m))
137 )
138 end
139 end
140 M
141
142 julia> macro m()
143 2
144 end
145 @m (macro with 1 method)
146
147 julia> M.f()
148 (1, 1, 2)
149 ```
150 With `@macroexpand` the expression expands where `@macroexpand` appears in the code (module `M` in the example).
151 With `macroexpand` the expression expands in the module given as the first argument.
152 """
153 macro macroexpand(code)
154 return :(macroexpand($__module__, $(QuoteNode(code)), recursive=true))
155 end
